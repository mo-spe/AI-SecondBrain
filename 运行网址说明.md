### 方式二：本地部署

如果需要本地部署测试，请按以下步骤操作：

#### 1. 环境要求

- 操作系统：Windows 10/11、Ubuntu 20.04+、macOS 10.15+
- Docker Desktop（Windows/macOS）或 Docker Engine（Linux）
- 至少4GB可用内存
- 至少10GB硬盘空间

#### 2. 安装Docker

**Windows/macOS**：
1. 下载Docker Desktop：https://www.docker.com/products/docker-desktop
2. 安装并启动Docker Desktop
3. 验证安装：
   ```bash
   docker --version
   docker-compose --version
   ```

**Linux (Ubuntu)**：
```bash
curl -fsSL https://get.docker.com | bash
curl -L "https://github.com/docker/compose/releases/download/v2.23.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose
```

#### 3. 部署步骤

**Windows**：
```powershell
# 解压安装包
Expand-Archive AI-SecondBrain-1.0.0.zip -DestinationPath .
cd AI-SecondBrain-1.0.0

# 运行部署脚本（以管理员身份运行PowerShell）
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
.\deploy.ps1
```

**Linux/macOS**：
```bash
# 解压安装包
tar -xzf AI-SecondBrain-1.0.0.tar.gz
cd AI-SecondBrain-1.0.0

# 运行部署脚本
chmod +x deploy.sh
./deploy.sh
```

#### 4. 访问系统

部署完成后，通过以下地址访问：

- **前端**：http://localhost
- **后端API**：http://localhost:8080
- **API文档**：http://localhost:8080/doc.html

**默认账号**：
- 用户名：testuser
- 密码：123456

### 方式三：使用云服务器

如果你有云服务器，可以按照以下步骤部署：

#### 1. 购买云服务器

推荐配置：
- CPU：2核
- 内存：4GB
- 硬盘：50GB SSD
- 带宽：5Mbps

推荐云服务商：
- 阿里云：https://www.aliyun.com
- 腾讯云：https://cloud.tencent.com
- 华为云：https://www.huaweicloud.com

#### 2. 配置安全组

开放以下端口：
- 80（HTTP）
- 443（HTTPS）
- 22（SSH）

#### 3. 部署项目

```bash
# SSH连接服务器
ssh root@your-server-ip

# 安装Docker
curl -fsSL https://get.docker.com | bash

# 上传并解压安装包
# 可以使用scp、ftp等方式上传

# 运行部署脚本
./deploy.sh
```

#### 4. 配置域名（可选）

如果有域名，可以配置域名访问：

1. 域名解析：添加A记录指向服务器IP
2. 修改配置：编辑 `.env` 文件，设置 `DOMAIN_NAME=your-domain.com`
3. 重启服务：`docker-compose restart`

#### 5. 配置HTTPS（可选）

```bash
# 安装Certbot
apt install certbot

# 申请证书
certbot certonly --standalone -d your-domain.com

# 复制证书
cp /etc/letsencrypt/live/your-domain.com/*.pem ssl/

# 启用HTTPS
# 修改 .env 文件：ENABLE_HTTPS=true
docker-compose restart nginx
```

## 🔧 常见问题

### 1. 无法访问localhost

**解决方案**：
- 确认Docker服务已启动
- 检查防火墙设置
- 尝试使用 `http://127.0.0.1`

### 2. 端口被占用

**解决方案**：
```bash
# 查看端口占用
netstat -ano | findstr :80
netstat -ano | findstr :8080

# 修改端口配置
# 编辑 .env 文件
FRONTEND_PORT=8081
BACKEND_PORT=8082
```

### 3. Docker启动失败

**解决方案**：
- Windows：确保Docker Desktop已启动
- Linux：运行 `systemctl start docker`
- 检查Docker版本：`docker --version`

### 4. 内存不足

**解决方案**：
- 关闭其他应用程序
- 调整Docker内存限制（Docker Desktop设置）
- 修改 `.env` 文件中的内存配置
