# AI-SecondBrain 概要设计

## 1. 设计概述

### 1.1 设计目标

AI-SecondBrain概要设计旨在构建一个高性能、高可用、可扩展的智能知识管理系统，实现以下目标：

1. **系统性能**：支持100+并发用户，API响应时间<200ms，搜索响应时间<500ms
2. **系统可用性**：系统可用性>99.5%，数据一致性100%
3. **可扩展性**：支持水平扩展，知识点数量>10000
4. **安全性**：密码加密存储，JWT认证，防止SQL注入和XSS攻击
5. **用户体验**：页面加载时间<2秒，操作流畅，界面美观

### 1.2 设计原则

| 原则         | 说明                                 |
| ------------ | ------------------------------------ |
| 分层架构     | 前端、后端、数据库分层，职责清晰     |
| 微服务化     | 核心功能模块化，便于独立部署和扩展   |
| 高内聚低耦合 | 模块内部高内聚，模块之间低耦合       |
| 可测试性     | 代码结构清晰，便于单元测试和集成测试 |
| 可维护性     | 代码规范，注释完整，易于维护和扩展   |
| 安全第一     | 安全设计贯穿始终，防止各种安全威胁   |

## 3. 核心流程设计

### 3.1 用户认证流程

```
用户输入用户名密码
    ↓
前端发送登录请求
    ↓
后端验证用户名密码
    ↓
生成JWT Token
    ↓
返回Token给前端
    ↓
前端存储Token
    ↓
后续请求携带Token
    ↓
后端验证Token
    ↓
允许访问或拒绝
```

### 3.2 知识提取流程

```
用户采集对话
    ↓
发送到Kafka消息队列
    ↓
消费者接收消息
    ↓
调用AI API提取知识点
    ↓
保存到MySQL数据库
    ↓
生成向量嵌入
    ↓
索引到Elasticsearch
    ↓
返回结果给用户
```

### 3.3 复习流程

```
系统计算待复习列表
    ↓
用户查看今日待复习
    ↓
用户选择复习卡片
    ↓
用户答题
    ↓
提交答案
    ↓
系统判断对错
    ↓
更新复习进度
    ↓
计算下次复习时间
    ↓
更新数据库
    ↓
返回复习结果
```

### 3.4 搜索流程

```
用户输入搜索关键词
    ↓
前端发送搜索请求
    ↓
后端接收请求
    ↓
根据搜索类型选择搜索方式
    ↓
关键词搜索 → MySQL模糊查询
    ↓
多字段搜索 → Elasticsearch全文搜索
    ↓
语义搜索 → Elasticsearch向量检索
    ↓
返回搜索结果
    ↓
前端展示结果
```

## 5. 安全设计

### 5.1 认证与授权

- **JWT认证**：使用JWT Token进行身份认证，有效期24小时
- **密码加密**：使用BCrypt算法加密存储密码
- **权限控制**：用户只能访问自己的数据

### 5.2 数据安全

- **SQL注入防护**：使用MyBatis Plus参数化查询
